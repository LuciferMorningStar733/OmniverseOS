You now have enough context of OmniverseOS. The desktop experience is production-quality and must remain visually identical. This task is ONLY focused on making the Android/mobile experience feel like it was designed as a native mobile operating system rather than a desktop website.

## Goal

Transform the mobile UX into something comparable to Android/iOS quality while preserving the OmniverseOS cyberpunk identity.

Do NOT redesign branding, colors, typography, or desktop layouts.

Desktop behavior must remain pixel-identical.

---

# MOBILE UX POLISH

Perform a complete UX audit of every mobile interaction.

Remove every source of jank, lag, awkward spacing or desktop-like behavior.

Target 60 FPS animations.

Avoid unnecessary React rerenders.

Optimize touch responsiveness.

Debounce expensive resize listeners.

Memoize where appropriate.

Use hardware accelerated transforms (translate3d, transform, opacity).

Avoid animating width/height where possible.

---

# APP HEADER

Every fullscreen mobile app must include a native header.

Required layout:

[← Back]    App Name                 [✕ Close]

Back returns to the previous screen.

Close immediately closes the app and returns to the desktop.

Do NOT rely only on gestures.

Gestures should remain as shortcuts, not the only navigation method.

Header buttons must be at least 44x44px touch targets.

No layout jumps.
